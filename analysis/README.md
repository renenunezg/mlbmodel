# Analysis

Read-only analysis scripts that interrogate the production model's track record.
**These scripts never write to the DB and never modify production model artifacts.**

## Layout

```
analysis/
  _common.py                  # shared DB connection + plot styling
  01_calibration.py           # raw vs isotonic-calibrated win prob calibration
  02_comeback_losses.py       # how often does our pick lose due to late-inning runs
  03_bullpen_fatigue.py       # edge realization vs opposing bullpen recent IP load
  outputs/                    # gitignored, regenerated by each script
    01_calibration/
    02_comeback_losses/
    03_bullpen_fatigue/
```

## Running

```bash
source env/bin/activate
pip install -e .              # one-time: makes `analysis` and `backend` importable
python analysis/01_calibration.py
```

Each script writes PNGs, CSVs, and a `summary.md` to its `outputs/` subdirectory.

## Package layout

The repo has a top-level `pyproject.toml` and is consumed via `pip install -e .`,
so analysis scripts can do `from analysis._common import ...` and
`from backend.db import engine` cleanly without `sys.path` hacks.
