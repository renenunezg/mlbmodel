# Analysis

Read-only helpers for ad-hoc analysis against the production DB.
**Code here never writes to the DB and never modifies production model artifacts.**

## Layout

```
analysis/
  _common.py        # shared DB connection + plot styling
  _linescores.py    # per-inning linescore helpers
  _boxscores.py     # per-game boxscore helpers
  outputs/          # gitignored, regenerated by any analysis script
```

## Package layout

The repo has a top-level `pyproject.toml` and is consumed via `pip install -e .`,
so analysis scripts can do `from analysis._common import ...` and
`from backend.db import engine` cleanly without `sys.path` hacks.
